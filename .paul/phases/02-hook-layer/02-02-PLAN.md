<objective>
## Goal
Ship the read-only `session-pulse` hook: a `SessionStart:startup` entrypoint that renders up to three injection tags — `<active-roster>`, `<pending-gates>`, `<goal-health>` — from the workspace's `.firm/firm.db` into every Claude Code session.

## Purpose
First visible surface of the Agent Company framework. Operators see their Firm (active Members, pending Gates, Goal health) at every session start without running a single command. Unlocks Phase 3 slash commands (they can assume the roster context is already present) and validates the `.firm/` → `.claude/hooks/` install path end-to-end before the rest of Phase 2 builds on it.

## Output
- `firm.hooks.session_pulse` Python module (pure renderer, SQL reads only)
- `firm.hooks.render` utilities (polymorphic name dispatcher, time-ago formatter, silent-when-empty helper)
- Hook entrypoint template `install/firm-session-pulse.py` + idempotent installer `install/hook-installer.py`
- Installed copy at `<workspace>/.claude/hooks/firm-session-pulse.py` registered in `.claude/settings.json`
- Unit tests per tag + seeded e2e test with golden output
- `.paul/phases/02-hook-layer/02-02-SUMMARY.md` after UNIFY
</objective>

  <name>Task 1: Implement `firm.hooks.session_pulse` renderer module</name>
  <files>src/firm/hooks/__init__.py, src/firm/hooks/session_pulse.py, src/firm/hooks/render.py, tests/hooks/__init__.py, tests/hooks/test_session_pulse.py</files>
  <action>
    Create `src/firm/hooks/` Python package. Three modules:

    1. `render.py` — pure utilities:
       - `resolve_entity_name(conn, entity_type: str, entity_id: str) -> str | None` — polymorphic dispatcher. Allowed entity_type values: firm, member, operation, project, unit, goal, document, firm_secret, contract (CHECK constraint in 002_entities.sql line 303). Returns `None` if row missing; caller renders `(target missing)` fallback.
       - `time_ago(ts_iso: str, now: datetime | None = None) -> str` — renders "3h ago", "1d ago", "5m ago" for an ISO second-resolution timestamp.
       - `classify_expiry(expires_at: str | None, now: datetime) -> Literal["EXPIRED", "URGENT", "STANDARD"]` — per BRIEF §2.2: `expires_at<now → EXPIRED`, `expires_at<now+24h → URGENT`, else STANDARD. NULL `expires_at` → STANDARD.

    2. `session_pulse.py` — three tag renderers + orchestrator:
       - `render_active_roster(conn, firm_id) -> str | None` — runs the SQL from BRIEF §2.1. Renders the `<active-roster members="N">` block, BOARD / MANAGERS / INDIVIDUAL CONTRIBUTORS sections, `CURRENTLY ON` line when `claimed_unit_id` present, `(no contract wired yet)` when `entry_command` is NULL. Returns `None` only when zero rows (no active Members) — otherwise always rendered.
       - `render_pending_gates(conn, firm_id) -> str | None` — SQL from §2.2 step 1, then per-row `resolve_entity_name` dispatch. Groups by `classify_expiry`. Returns `None` when zero pending Gates (silent-when-empty per decision 6 in BRIEF §6).
       - `render_goal_health(conn, firm_id) -> str | None` — SQL from §2.3. Handles null `metric.value` → renders `"target null {unit}"`, null `metric.current` → `"current not-yet-baselined"`. Computes overdue line ONLY when `metric.deadline` present. Returns `None` when zero active Goals.
       - `render(conn, firm_id) -> str` — calls all three, concatenates non-None outputs with a blank line between, returns `""` when all three are None (hook entrypoint prints nothing).

    3. `__init__.py` — package marker, no re-exports (callers use full paths).

    Read-only SQL only. No DB writes. Use `firm.core.repo` or direct cursor; transaction not required for reads.

    Tests in `tests/hooks/test_session_pulse.py`:
    - Unit test each renderer with a seeded in-memory DB fixture (reuse Phase 1 test fixture pattern from `tests/test_repo.py`).
    - One test per AC-1, AC-2 (both empty-set and 3-gate cases), AC-3.
    - Test `resolve_entity_name` returns None when target row missing; render_pending_gates falls back to `(target missing)`.
    - Test `classify_expiry` edge cases: exactly at boundary (expires_at == now), NULL, 23h59m ahead, 24h1m ahead.
  </action>

<task type="auto">
  <name>Task 2: Hook entrypoint + idempotent installer</name>
  <files>install/firm-session-pulse.py, install/hook-installer.py</files>
  <action>
    Two artifacts:

    1. `install/firm-session-pulse.py` — standalone entrypoint script (lives in repo as template; installer copies to `<workspace>/.claude/hooks/`):
       - Shebang `#!/usr/bin/env python3`.
       - Reads stdin JSON (`{session_id, cwd, ...}` per Claude Code SessionStart spec).
       - Resolves workspace from `cwd` field (Path).
       - Resolves `FIRM_ID = os.environ.get("FIRM_ID", "chrisai")`.
       - Adds `<workspace>/src` to sys.path so `firm` package is importable when installed side-by-side with the agent-company-architecture repo. If import fails (fresh workspace without the repo), silent `sys.exit(0)`.
       - Opens connection via `firm.core.db.db_connection(workspace)`; if `.firm/firm.db` missing, silent `sys.exit(0)`.
       - Calls `firm.hooks.session_pulse.render(conn, firm_id)`.
       - Prints result to stdout (empty string → prints nothing).
       - Wraps everything in `try/except Exception: sys.exit(0)` — hook NEVER blocks session start (§3.1 "Exit code: 0 always").

    2. `install/hook-installer.py` — idempotent installer utility:
       - CLI: `python install/hook-installer.py <workspace-path>`.
       - Creates `<workspace>/.claude/hooks/` if missing.
       - Copies `install/firm-session-pulse.py` → `<workspace>/.claude/hooks/firm-session-pulse.py`, `chmod +x`.
       - Reads `<workspace>/.claude/settings.json` (creates empty `{}` if missing). Navigates/creates `hooks.SessionStart` array. Appends an entry with `matcher: "startup"` and `hooks: [{ type: "command", command: "python3 $CLAUDE_PROJECT_DIR/.claude/hooks/firm-session-pulse.py" }]` — but ONLY if no existing entry with the same command string is present (idempotency check).
       - Writes settings.json back with 2-space indent preserved.
       - Exits 0 with a one-line summary `"Installed firm-session-pulse → <path>"` or `"Already installed at <path> — no changes."`.

    Mirror BASE installer shape if one exists in `.base/` tree. Do NOT introduce any new runtime deps (stdlib only: json, pathlib, shutil, os, sys).
  </action>
