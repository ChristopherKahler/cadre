## AC-1: `on_run_end` finalizes member_run and writes immutable usage_event atomically
```gherkin
Given a running member_run row (status='running') with a valid member_id and firm_id
When on_run_end(workspace, run_id, final_status='completed', outputs=[...], usage={...}, notes=None, error=None) is called
Then member_run.status is updated to 'completed', ended_at is set, outputs/error/notes persist
And a new usage_event row is inserted (immutable) with all token fields populated from the usage dict
And a new records row is inserted for the 'member_run.ended' event with LOG-NNN id
And all three writes commit in a single transaction (or none commit)
```

## AC-2: When run has a unit_id, unit.outputs is merged with run.outputs as the fourth write
```gherkin
Given a member_run with unit_id set (Run was Unit-scoped)
When on_run_end is called with final_status='completed' and outputs=[...]
Then unit.outputs JSON is updated to include the new run outputs (existing items preserved)
And the unit.outputs UPDATE participates in the same transaction as the other three writes

Given a member_run with unit_id=NULL (Run not Unit-scoped)
When on_run_end is called
Then no unit row is touched (3 writes instead of 4)
```

## AC-3: Partial failure rolls back all writes — no orphan state
```gherkin
Given on_run_end is invoked and the third write (records INSERT) raises a sqlite3 error
When the transaction is rolled back
Then member_run.status still reads 'running' (original value)
And no usage_event row exists for this run_id
And no records row exists for this run_id
And unit.outputs still reads its pre-call value (if unit_id present)
```

## AC-4: Credential redaction strips secret-shaped keys from error and notes before write
```gherkin
Given an error dict {"message": "auth failed", "api_key": "sk-abc123"}
When on_run_end is called with error=<that dict>
Then the persisted member_run.error has "api_key" replaced with "[REDACTED]"
And the same redaction applies to notes (string scan — lines matching /(?i)(token|key|secret|password)[\s:=]+\S+/ get their value replaced)
And the original error dict passed by the caller is NOT mutated (redaction is on a copy)
```

## AC-5: `firm run end <run_id>` CLI verb invokes on_run_end with full flag parity
```gherkin
Given a workspace with .firm/firm.db containing a running member_run RUN-001
When the user runs `firm run end RUN-001 --status completed --outputs '[{"path":"post.md"}]' --workspace <path>`
Then on_run_end is called with parsed arguments, DB writes commit, exit 0
And `firm run end RUN-001 --dry-run` prints what would change (row counts pre/post) without committing
And `--firm-id <id>` flag takes precedence over FIRM_ID env var
And run-not-found returns {ok: false, reason: "run-not-found"} with exit 0 (not a crash)
```

  <action>
    **Start from the raw-SQL transaction pattern established in 02-03.** `repo.create` / `repo.update` commit internally — USE RAW SQL inside a single manual `try/commit/except+rollback` block for all 4 writes. This is amplified vs 02-03 (4 rows vs 2).

    **`_redact.py`** (~20 LOC):
    - Export `redact(value: Any) -> Any` — pure function, returns redacted copy.
    - For dicts: walk recursively, replace any value whose key matches `/(?i)(token|key|secret|password)/` with `"[REDACTED]"`. Leave other keys alone.
    - For strings: regex-replace lines matching `/^(\s*)([A-Za-z_][\w-]*(?:token|key|secret|password)[\w-]*)(\s*[:=]\s*)(\S+)/im` with group 1+2+3+`[REDACTED]`. Also handle inline `token=abc123` patterns.
    - For lists: redact each item.
    - For other types: return as-is.
    - **Do NOT mutate input** — always return a new structure.

    **`run_record.py`** (~150 LOC):
    - Signature per BRIEF §3.3:
      `on_run_end(workspace: Path, run_id: str, *, final_status: str, outputs: list | None = None, usage: dict | None = None, notes: str | None = None, error: dict | None = None, now: datetime | None = None) -> dict`
    - Accepts optional `now` for deterministic testing (follows FIRM_NOW_OVERRIDE convention from 02-02).
    - Returns summary dict: `{ok: bool, run_id, wrote: {member_run, usage_event, unit, records}, records_id, reason?}`.
    - Sequence inside single manual transaction (`conn.execute("BEGIN")` → writes → `conn.commit()` on success / `conn.rollback()` on any exception):
      1. Fetch `member_run` row by id — if not found, return `{ok: false, reason: "run-not-found"}` WITHOUT opening a transaction. Capture `member_id`, `firm_id`, `unit_id`, `started_at`.
      2. `BEGIN`.
      3. **Write 1** — `UPDATE member_run SET status=?, ended_at=?, outputs=?, error=?, notes=? WHERE id=?` with redacted error/notes via `_redact.redact(...)`.
      4. **Write 2** — `INSERT INTO usage_event` (id=`USG-<next>`, firm_id, member_id, run_id, tokens from usage dict — partial-OK with nulls if usage is None, created_at). id scheme: `USG-NNN` via `SELECT COUNT(*) FROM usage_event WHERE firm_id=?` + 1 (mirrors LOG-NNN scheme from 02-03; same concurrency caveat documented).
      5. **Write 3 (conditional)** — if `unit_id` present: `UPDATE unit SET outputs=? WHERE id=?` merging new outputs with existing (append-style, JSON array concat). Skip if unit_id NULL.
      6. **Write 4** — `INSERT INTO records` (id=`LOG-<next>`, event_type=`member_run.ended`, details=JSON with run_id/final_status/outputs_count/had_error, created_at). Reuse `_next_records_id` from `unit_completion.py` (import it).
      7. `COMMIT`.
    - On `sqlite3.Error` during any write: `ROLLBACK`, re-raise as-is so caller sees the real error.
    - **Export `on_run_end` in `__init__.py`** alongside existing `on_unit_done`.

    **`__init__.py`** update:
    - Add `from firm.hooks.run_record import on_run_end`.
    - Append to `__all__`.

    **`tests/hooks/test_run_record.py`** (~15 tests):
    - Fixture: seeded workspace DB with Quill, Sterling, Sage, OPS-001, a project, a unit (UNIT-001 with unit_id), and a running member_run (RUN-001 linked to unit_id, RUN-002 NOT linked to unit_id).
    - Tests:
      1. `test_completes_run_with_full_usage` — happy path, 4 writes, row counts verified.
      2. `test_completes_run_without_unit_id` — RUN-002, 3 writes only (no unit UPDATE).
      3. `test_run_not_found_returns_reason` — `on_run_end(..., run_id="RUN-999")` returns `{ok: false, reason: "run-not-found"}`, no writes.
      4. `test_failed_run_persists_error` — `final_status='failed'`, error dict persists.
      5. `test_redacts_api_key_from_error` — error contains `api_key`, persisted member_run.error shows `[REDACTED]`.
      6. `test_redacts_token_from_notes` — notes contains `auth_token=abc123`, persisted notes has `[REDACTED]`.
      7. `test_redaction_does_not_mutate_input` — caller's error dict unchanged after call.
      8. `test_usage_event_nulls_when_usage_missing` — no usage dict, usage_event row exists with null token fields.
      9. `test_usage_event_has_all_token_fields` — full usage dict, all fields populated.
      10. `test_unit_outputs_merged_not_replaced` — unit already has outputs `[{"old": 1}]`, after call has `[{"old": 1}, ...new]`.
      11. `test_records_row_has_correct_event_type` — `event_type='member_run.ended'`.
      12. `test_records_id_increments` — second call produces LOG-(N+1).
      13. `test_mid_transaction_rollback` — use DB-level trigger (copy pattern from `test_unit_completion.py::test_mid_transaction_rollback`) to force `usage_event` INSERT to raise. Verify: member_run.status still 'running', no usage_event row, no records row, unit.outputs unchanged.
      14. `test_run_id_threading_via_records_details` — details JSON has run_id.
      15. `test_now_override_deterministic` — `on_run_end(..., now=fixed_dt)` produces exact ended_at and created_at values.
