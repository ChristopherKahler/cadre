<objective>
## Goal
Produce a research brief that designs the complete `firm` Claude Code skill following Skillsmith conventions, maps every slash command to its entity lifecycle operation, defines the Python service layer bridge, and scopes the execute plans for 03-02 through 03-NN.

## Purpose
Phase 3 is where the framework becomes interactive. Phase 1 gave us the data substrate (SQLite, 14 entity tables, CRUD repo). Phase 2 gave us session-time hooks (session-pulse, unit-completion, run-record). Phase 3 gives the operator and Members a command surface to create, view, update, and lifecycle-manage every entity without raw SQL or Python imports.

The operator has mandated that this phase follow `/skillsmith:skillsmith` conventions. This means:
- Slash commands live as Skillsmith-compliant skill files (entry point + tasks + frameworks + context + checklists)
- The skill architecture (XML sections, routing, naming) must pass a Skillsmith audit
- The Python backend serves as the execution engine; skill task files orchestrate it

Locking the skill spec and command surface design now prevents rework across 10+ task files.

## Output
`.paul/phases/03-core-slash-commands/03-01-BRIEF.md` containing:
1. **Complete command surface map** - every `/firm:*` command, its sub-actions, arguments, entity operations, and read/write profile
2. **Skillsmith skill spec** - filled-out skill-spec template (identity, persona, activation, commands, content architecture) ready for `/skillsmith scaffold`
3. **Python service layer design** - function signatures for `firm.services.*` modules that bridge skill tasks to `firm.core.repo`
4. **Scope decisions** - which entity commands are Phase 3 vs deferred (with rationale)
5. **Execute plan breakdown** - recommended split for 03-02 through 03-NN with task outlines and wave assignments
6. **Decision log entries** to record in PROJECT.md
</objective>
